# NCBlog (Netease/Nova Creator Blog)

[![License](https://img.shields.io/github/license/username/project.svg)](LICENSE)
[![PHP Version](https://img.shields.io/badge/php-%3E%3D%207.4-8892bf.svg)](https://www.php.net/)
[![MySQL Version](https://img.shields.io/badge/mysql-%3E%3D%205.7-blue.svg)](https://www.mysql.com/)

**NCBlog** 是一款基于原生 PHP + MySQL 开发的轻量级、高颜值、功能完善的自媒体与内容付费订阅博客系统。它专为个人创作者、自媒体运营者设计，不仅提供极致的阅读体验，还深度集成了**会员订阅、单篇付费、AI 辅助写作、SEO 主动推送**等多维度功能，助您轻松实现内容变现。

---

## 🎨 界面预览

*（您可以将您本机的运行截图上传并在此处替换图片链接）*

- **PC端首页**: 采用现代化毛玻璃微动画设计，侧边栏聚合分类、标签、最新评论。
- **文章详情**: 优雅的排版，支持 Markdown 或富文本渲染，包含单篇付费解锁组件。
- **订阅中心**: 黄金比例订阅卡片（月度/年度套餐），支持易支付扫码即时开通。
- **后台管理**: 响应式多功能面板，包含数据看板、文章/分类管理、支付接口配置、SEO 推送、AI 助手等模块。

## 1. 系统环境要求

在部署 NCBlog 之前，请确保您的服务器环境满足以下要求：

- **操作系统**：Linux (如 CentOS / Ubuntu / Debian，推荐) 或 Windows
- **Web 服务器**：Nginx (推荐) 或 Apache
- **PHP 版本**：7.4 至 8.2（推荐 PHP 8.0/8.1）
- **MySQL 版本**：5.7 或 8.0+
- **PHP 扩展要求**：
  - `pdo_mysql` (数据库连接)
  - `curl` (用于 AI API 请求、易支付及 SEO 推送)
  - `openssl` (安全加密与通信)
  - `mbstring` (多字节字符处理)
  - `fileinfo` (用于附件和图片上传类型识别)

## 2. 伪静态 (URL Rewrite) 配置

NCBlog 采用现代化的 URL 路由设计（例如 `/article/my-first-post`）。您必须配置 Web 服务器的伪静态，使所有非静态文件请求都指向根目录下的 `index.php`。

### 🔹 Nginx 配置
在您的 Nginx 站点配置文件（通常位于 `nginx.conf` 或 `vhost.conf`）的 `server` 块中加入以下规则：

```nginx
location / {
    try_files $uri $uri/ /index.php?$query_string;
}

# 拒绝直接访问敏感目录和文件
location ~ ^/(data|includes|sql)/ {
    deny all;
}
location ~ /\.(env|git|ht) {
    deny all;
}
```

### 🔹 Apache 配置
项目根目录已附带 `.htaccess` 文件，通常无需手动配置。如果您的 Apache 服务器未生效，请确保 `httpd.conf` 启用了 `mod_rewrite` 模块，并允许 `.htaccess` 覆写：
```apache
AllowOverride All
```

## 3. 一键安装指南

1. **上传代码**：将整个 `ncblog` 项目文件上传至网站根目录。
2. **设置目录权限**（Linux 环境下）：
   确保 Web 运行用户（如 `www` 或 `nginx`）对以下目录拥有**读写权限**：
   ```bash
   chmod -R 755 /www/wwwroot/your-site/
   chmod -R 777 /www/wwwroot/your-site/data/
   chmod -R 777 /www/wwwroot/your-site/config.php
   ```
3. **开始安装向导**：
   - 浏览器访问：`http://你的域名/install.php`。
   - 输入您的数据库连接信息（主机名、端口、数据库名、用户名、密码）。
   - 设置您的管理员后台账号、密码和安全邮箱。
   - 点击 **一键安装**。
4. **验证安装**：
   - 安装完成后，前台首页可通过：`http://你的域名/` 访问。
   - 后台管理地址为：`http://你的域名/admin/login.php`。
   - 安装成功后，系统会自动在根目录下创建 `data/install.lock` 文件以锁定安装程序。

## 4. 💰 订阅与付费设置 (对接易支付)

NCBlog 支持单篇内容付费购买与全局会员订阅服务，默认对接标准易支付（Epay）接口：

1. **获取易支付商户信息**：
   - 在任何主流的易支付平台注册并获取：**商户 ID (PID)**、**商户密钥 (KEY)** 以及 **API 支付网关地址**。
2. **后台配置**：
   - 登录 NCBlog 后台 -> 侧边栏导航点击 **支付配置 / 订阅设置** (`admin/page_payment.php`)。
   - 填入您的商户 ID、商户密钥以及支付网关接口。
   - 设置订阅套餐价格（如：月度会员价格 9.9 元，年度会员价格 99 元）。
3. **功能测试**：
   - 前台点击“订阅会员”或在付费文章中点击“立即解锁”。
   - 系统将自动生成订单，跳转到易支付收银台。
   - 扫码支付成功后，通过异步通知（`api/pay/notify`），系统会自动将对应邮箱/用户升级为有效订阅用户，无需人工干预。
   - 若您未配置易支付，用户支付时可选择手动转账，由您在后台 **订阅管理** 中手动审核开通。

## 5. 📧 SMTP 邮件发送配置

注册账户需要发送验证码，评论审核也需要通知站长。请正确配置邮件系统：

1. **获取 SMTP 信息**：
   - 登录您的邮箱后台（如 QQ 邮箱、网易 163 邮箱、Gmail、阿里云邮等），开启 **SMTP 服务** 并生成 **授权码 / 客户端密码**。
2. **后台配置**：
   - 登录后台 -> 点击 **邮件服务设置** (`admin/page_email.php`)。
   - 填写以下参数：
     - **SMTP 服务器地址** (如 `smtp.qq.com`)
     - **SMTP 端口** (通常 SSL 为 `465`，非加密为 `25` 或 `587`)
     - **发件人邮箱** (您的完整邮箱地址)
     - **客户端授权密码** (非邮箱登录密码，而是生成的授权码)
     - **发件人昵称** (前台显示的发件人名称，如：NCBlog 官方团队)
3. **发送测试**：
   - 在邮件配置页面点击“发送测试邮件”，填入您的接收邮箱，收到测试邮件即代表配置成功。

## 6. 🤖 AI 创作助手配置

后台集成了基于 LLM（大语言模型）的 AI 写作助手，能根据您的提示词或大纲，在线一键生成文章内容。

1. **后台配置**：
   - 登录后台 -> 点击 **AI 写作助手** (`admin/page_ai_writer.php`)。
   - 选择大模型提供商（如：OpenAI, Claude, 百度千帆, 腾讯混元, 阿里云通义千问，或标准 OpenAI 兼容格式网关）。
   - 填写 API Key 及选择要调用的具体模型模型名（如 `gpt-4o`, `deepseek-chat` 等）。
2. **使用 AI 创作**：
   - 在后台发布文章时，点击编辑框旁边的 **AI 智能生成**。
   - 输入您想要创作的标题或灵感词，AI 将在数秒内输出高质量草稿，并支持一键导入到编辑器。

## 7. 📈 SEO 自动推送设置

NCBlog 拥有极佳的搜索引擎友好性，支持自动和手动将新发布的文章链接推送到各大搜索引擎。

1. **配置 sitemap**：
   - 系统会自动在 `http://你的域名/sitemap.xml` 生成最新的网站地图，您可以直接将此链接提交到百度搜索资源平台和 Google Search Console。
2. **配置链接主动推送**：
   - 登录后台 -> 点击 **SEO 优化配置** (`admin/page_seo_push.php`)。
   - 填写您的 **百度 API 准入密钥 (token)** 或配置 Bing IndexNow。
   - 之后每次在后台点击“发布新文章”时，系统会在后台悄悄把新链接自动推送给搜索引擎，缩短收录时间。

## 8. 🔒 系统安全建议

为了确保您的网站数据和财务安全，建议在正式上线后执行以下操作：

1. **隐藏后台入口**：
   - 可以重命名 `admin/` 目录名称（如修改为 `my_secret_admin/`），并在代码中保持路径引用一致。
2. **删除或锁定安装脚本**：
   - 检查 `data/install.lock` 是否存在，如果不存在，请手动创建一个空文件并重命名为 `install.lock` 放入 `data` 文件夹内，防止他人重新运行 `install.php` 覆盖数据库。
3. **开启防刷防爬**：
   - 后台的 **安全设置** 中，可限制单 IP 的日请求频率，并对频繁恶意刷页面的 IP 自动加入临时黑名单。
